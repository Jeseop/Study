22.02.22 4장 변수 컴퓨터는 CPU를 사용해 연산, 메모리를 사용해 데이터를 기억함. 메모리는 데이터를 저장할 수 있는 메모리 셀의 집합체. 셀 하나의 크기는 1바이트 (8비트). 컴퓨터는 1바이트 단위로 데이터를 저장하거나 읽음. 각 셀은 고유의 메모리 주소를 갖음. 메모리 주소 : 메모리 공간의 위치. 메모리 주소는 메모리의 크기만큼 정수로 표현됨. 컴퓨터는 모든 데이터를 '2진수'로 처리함. 예를 들어 10+20이라는 식이 있다고 가정해보자. 10과 20은 피연산자로써 메모리 상의 임의의 위치에 기억된다. CPU가 연산하고 30이라는 결과값은 메모리 주소에 저장된다. 그러나, 재사용이 불가하다. 만약 '30'을 재사용하고 싶다면 메모리 주소를 통해 메모리 공간에 직접 접근하는 것 외에 방법이 없다. 그렇지만 이것이 치명적인 오류를 발생시킬 가능성이 높기에 '매우 위험한 일'이다. 그래서 자바스크립트는 개발자의 직접적인 메모리 제어를 허용하지 않는다. 메모리 주소를 통해 값에 직접 접근하려는 시도는 올바른 방법이 아니다. 프로그래밍 언어는 기억하고 싶은 값을 메모리에 저장하고, 저장된 값을 읽어 들여 재사용하기 위해 변수라는 메커니즘을 제공한다.
