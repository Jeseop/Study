22.02.22 4장 변수 4.1 변수란 무엇인가/ 왜 필요한가? 컴퓨터는 CPU를 사용해 연산, 메모리를 사용해 데이터를 기억함. 메모리는 데이터를 저장할 수 있는 메모리 셀의 집합체. 셀 하나의 크기는 1바이트 (8비트). 컴퓨터는 1바이트 단위로 데이터를 저장하거나 읽음. 각 셀은 고유의 메모리 주소를 갖음. 메모리 주소 : 메모리 공간의 위치. 메모리 주소는 메모리의 크기만큼 정수로 표현됨. 컴퓨터는 모든 데이터를 '2진수'로 처리함. 예를 들어 10+20이라는 식이 있다고 가정해보자. 10과 20은 피연산자로써 메모리 상의 임의의 위치에 기억된다. CPU가 연산하고 30이라는 결과값은 메모리 주소에 저장된다. 그러나, 재사용이 불가하다. 만약 '30'을 재사용하고 싶다면 메모리 주소를 통해 메모리 공간에 직접 접근하는 것 외에 방법이 없다. 그렇지만 이것이 치명적인 오류를 발생시킬 가능성이 높기에 '매우 위험한 일'이다. 그래서 자바스크립트는 개발자의 직접적인 메모리 제어를 허용하지 않는다. 메모리 주소를 통해 값에 직접 접근하려는 시도는 올바른 방법이 아니다. 프로그래밍 언어는 기억하고 싶은 값을 메모리에 저장하고, 저장된 값을 읽어 들여 재사용하기 위해 변수라는 메커니즘을 제공한다. 변수란? 변수는 하나의 값을 지정하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름. 간단하게 변수는 프로그래밍 언어에서 값을 저장하고 참조하는 매커니즘으로, 값의 위치를 가리키는 상징적인 이름 변수 이름 : 메모리 공간에 저장된 값을 식별할 수 있는 고유한 이름 변수 값 : 변수에 저장된 값 할당 : 변수에 값을 저장하는 것 Assignment(대입, 저장) 참조 : 변수에 저장된 값을 읽어 들이는 것 변수는 가독성을 높이는 효과를 지녔기에 심사숙고해서 지어야한다. 4.2 식별자 변수 이름 = 식별자 : 어떤 값을 구별해서 식별할 수 있는 고유한 이름 값은 '메모리 공간'에 저장. 식별자는 '메모리 주소'를 기억(저장)해야한다. ------------------- var result = 10+20; //= undefined// result //30// ------------------- result(식별자)는 30이 저장되어 있는 메모리 주소를 기억해야한다. 즉, 식별자는 (값이 저장되어 있는) 메모리 주소와 매핑 관계를 맺으며, 이 매핑 정보도 메모리에 저장되어야 한다. ※ Mapping(=사[본뜨다, 그리다]상[모양, 모습])이란? 해당 값이 다른 값을 가리키도록 하는 것. 지도에 표시한 정보가 현실 세계와 1:1로 대응하는 것처럼, 하나의 값을 다른 값으로 1:1 대응시키는 것. 이처럼 식별자는 값이 아닌 메모리 주소를 기억하고 있다. 식별자는 메모리 주소에 붙인 이름. 식별자라는 용어는 변수 이름에만 국한해서 사용하지 않음. ex) 변수, 함수, 클래스 등 메모리 상에 존재하는 어떤 값을 식별할 수 있는 이름은 모두 '식별자' 식별자는 네이밍 규칙을 준수해야한다. 선언(declaration)에 의해 자바스크립트 엔진에 식별자의 존재를 알린다. 4.3 변수 선언 변수 선언(Variable declaration) : 변수를 생성하는 것 값을 저장하기 위해 메모리 공간 확보 / 변수 이름과 확보된 메모리 공간의 주소를 연결(네임 바인딩)해 값의 저장 준비 변수 선언에는 반드시 '선언'이 필요. var, let, const 키워드를 사용. ES6에서 let과 const 키워드를 도입한 이유는 var 키워드의 여러 단점을 보안하기 위해. 그렇다면 var 키워드의 단점을 정확히 이해해야한다. 이해하기 위해선 자바스크립트의 핵심 개념을 먼저 살펴봐야한다. ES6에서 let과 const가 나왔다고 해서 var가 아예 폐기된 것은 아니다. ES6는 기본적으로 하위 호환성을 유지하면서 ES5의 기반위에 새로운 기능을 추가한 것이다. 다시 말하자면 ES6는 ES5의 상위 집합이다. ex) var score; -> score라는 이름으로 새로운 변수를 선언 1) 변수 이름을 등록하고 (선언 단계) 값을 저장할 메모리 공간 확보 (초기화 단계) 2) 변수를 선언한 이후, 아직 변수에 값을 할당하지 않음 3) 따라서 변수 선언에 의해 확보된 메모리 공간은 비어있는 것이 아닌 undefined라는 값이 암묵적으로 할당되어 초기화 됨. ※ 자바스크립트 엔진이 변수를 관리할 수 있도록 변수의 존재를 알리는 것이 포인트. 초기화란 변수가 선언된 이후 최초로 값을 할당하는 것을 말한다. var 키워드로 선언한 변수는 undefined로 암묵적인 초기화가 자동으로 수행된다. 초기화 단계를 안 거치면 쓰레시 값(이전에 다른 어플리케이션이 사용했던 값)이 남아 있을 수 있다. 선언하지 않은 식별자에 접근하려면 Reference Error(참조 에러)가 발생한다. 참조 에러 : 식별자를 통해 값을 참조하려 했지만 자바스크립트 엔진이 등록된 식별자를 찾을 수 없을 때 발생하는 에러 4.4 변수 선언의 실행 시점과 변수 호이스팅 1) console.log(score); // undefined -> 변수를 참조하는 코드 2) var score; // 변수 선언문 2보다 1이 앞에 있을 때 2가 안되서 참조에러가 발생할 것처럼 생각이 드는 사람들이 있을 것 같다. 하지만 undefined. 왜냐하면 변수 선언이 소스코드가 한줄씩 순차적으로 실행되는 시점, 즉 런타임이 아니라 그 이전 단계에서 먼저 실행되기 때문이다. 자바스크립트 엔진은 코드를 한줄씩 실행하기 전, 코드를 평가하는 과정을 거치며 실행할 준비를 한다. 소스 코드를 평가할 때 자바스크립트 엔진은 변수선언을 포함한 모든 선언문(변수 선언문, 함수 선언문 등)을 코드 내에서 찾아내 먼저 실행한다. 그리고 소스 평가 과정 끝나면 비로소 모든 선언문을 제외하고 코드를 한줄씩 순차 실행. '변수 선언문이 코드의 선두로 끌어올려진 것처럼 동작하는 자바스크립트 고유의 특징을 변수 호이스팅'이라한다. 4.5 값의 할당 할당하다(assign)하다는 대입하다 또는 저장하다라는 뜻. 변수에 값을 할당할 땐 할당 연산자 '='를 사용한다. 할당 연산자는 우변의 값을 좌변의 변수에 할당한다. 자바스크립트 엔진은 변수 선어노가 값을 할당을 하나의 문으로 단축표현해도 변수 선언과 값의 할당을 2개의 문(statement)으로 나누어 각각 실행한다. 변수 선언과 값의 할당 실행시점은 다르다는 것을 꼭 기억하자. 변수 선언은 소스코드가 순차적으로 실행되는 시점인 런타임 이전에 먼저 실행되지만 값의 할당은 소스코드가 순차적으로 실행되는 시점인 런타임에 실행된다. 4.6 값의 재할당 재할당이란? 이미 값이 할당되어 있는 변수에 새로운 값을 또 다시 할당하는 것 var number = 9; // 변수 선언과 값의 할당 number = 10; // 값의 재할당 var 키워드로 선언한 변수는 값을 재할당할 수 있다. (사실 var 키워드로 선언한 변수는 선언과 동시에 undefined로 초기화되기에 엄밀히 말하면 변수에 처음으로 값을 할당하는 것도 사실 재할당이다.) 값을 재할당할 수 없어서 변수에 저장된 값을 변경할 수 없다면 변수가 아닌 상수라고 한다. const 키워드를 사용하면 상수를 표현할 수 있다.(하지만 반드시 상수만을 위해 사용하지는 않는다고 한다. 추후에 다시 Deep하게 공부하게 된다.) 변수의 이전 값(들)은 어떤 변수도 값으로 갖고 있지 않다. 이러한 불필요한 것들은 가비지 콜렉터에 의해 메모리에서 자동 해제된다. ★가비지 콜렉터는 애플리케이션이 할당한 메모리 공간을 주기적으로 검사하여 더 이상 사용되지 않는 메모리를 해제(release)하는 기능. 자바스크립트는 가비지 콜렉터를 내장하고있는 매니지드 언어. 프로그래밍 언어는 '메모리 관리 방식'에 따라 언매니지드 언어와 매니지드 언어로 분류할 수 있다. 언매니지드 언어는 개발자의 역량에 따라 최적의 성능을 확보할 수 있지만, 치명적인 오류를 생산할 가능성도 있다. 매니지드 언어는 일정한 생산성을 확보할 수 있지만, 성능 면에서는 어느 정도 손실을 감수해야한다. 4.7 식별자 네이밍 규칙 식별자 : 어떤 값을 구별해서 식별해낼 수 있는 고유한 이름 - 식별자는 특수문자를 제외한 문자, '숫자', 언더스코어(_), 달러기호($)를 포함할 수 있다. - 단, 식별자는 특수문자를 제외한 문자, 언더스코어(_), 달러기호($)로 시작해야한다. 숫자 시작 불허용. - 예약어는 식별자로 사용할 수 없음, 예약어란 프로그래밍 언어에서 사용되고 있거나 사용될 예정인 단어. 변수도 식별자이므로 네이밍 규칙을 따라야한다. 변수는 쉼표(,)로 구분해 하나의 문에서 여러 개를 한번에 선언할 수 있음. ES5부터 한글이나 일본어도 사용하긴 하지만 비추천. 자바스크립트는 대소문자를 구별한다. hongjeseop, hongJeseop, Hongjeseop 다 다르다. 변수 이름은 변수가 왜 존재하는지 쉽고 명확하게 표현해야한다. 가독성을 높일 수 있기 때문. 변수 선언에 별도 주석이 필요하다면 망한 것이다. 네이밍 컨벤션은 하나 이상의 영어 단어로 구성된 식별자를 만들 때 가독성 좋게 단어를 한 눈에 구분하기 위해 규정한 명명 규칙. 1) 카멜 케이스 hongJeseop 2) 스네이크 케이스 hong_jeseop 3) 파스칼 케이스 HongJeseop 4) 헝가리언 케이스 : 접두어에 자료형을 알아볼 수 있도록 표시 일반적으로 변수나 함수의 이름에는 카멜케이스, 생성자 함수나 클래스 이름은 파스칼케이스를 사용한다. 따라서 코드 전체의 가독성을 높이려면 카멜 케이스와 파스칼 케이스를 따르는 것이 유리하다.
